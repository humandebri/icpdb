# ICPDB Goal Audit

This document maps the product goal to current evidence.

## Product Goal

Build an Internet Computer SQLite DBaaS:

- storage: `ic-sqlite-vfs`
- UI feel: Supabase Table Editor
- CLI/API feel: Turso shell and inspect

The current goal is ordinary SQL DB usability on IC, in this order:

1. SDK that feels like a normal SQL database client
2. shortest DB creation to `query` / `execute` path
3. console schema, table, and SQL inspection
4. CLI and `service.env` operation from Server/CI
5. archive/restore and shard operation
6. auth and permission polish

Full Turso compatibility is not a goal. ICPDB only keeps the useful SQL-client
shape (`createClient`, `execute`, `batch`, rows/result metadata, and migration
type aliases) while preserving IC identities, `icpdb://` URLs, principal ACLs,
and canister execution boundaries.
